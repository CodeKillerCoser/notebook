LeetCode Two Sum — Rust 实现与知识点分析

Two Sum 是 LeetCode 第 1 题：给定数组和目标值，返回两个下标使得其对应值相加等于目标值。

原始代码（包含错误）

impl Solution {
    pub fn two_sum(nums: Vec<i32>, target: i32) -> Vec<i32> {
        let len = nums.len();
        let mut index_arr = (0..len).collect();
        index_arr.sort_by(|a, b| nums[a].cmp(nums[b]));
        let mut i = 0;
        let mut j = len - 1;
        while i < j {
            match (nums[index_arr[i]] + nums[index_arr[j]]).cmp(&target) {
                std::cmp::Ordering::Less => i = i + 1,
                std::cmp::Ordering::Greater => j = j - 1,
                std::cmp::Ordering::Equal => {
                    return {index_arr[i], index_arr[j]};  // ❌ 语法错误
                }
            }
        }
        return {}  // ❌ 语法错误
    }
}

错误分析

错误 1：返回值语法错误

// ❌ {a, b} 不是合法的 Rust 表达式
return {index_arr[i], index_arr[j]};

// ✅ vec![a, b] 创建 Vec
return vec![index_arr[i] as i32, index_arr[j] as i32];

// ✅ 空返回值
vec![]

{a, b} 会被解释为块表达式（block expression），在 Rust 中不合法。创建 Vec 的正确方式是用 vec! 宏。

错误 2：类型不匹配

// ❌ index_arr 是 Vec<usize>，返回值需要 Vec<i32>
return {index_arr[i], index_arr[j]};

// ✅ 用 as 做类型转换
vec![index_arr[i] as i32, index_arr[j] as i32]

(0..len).collect() 根据后续用法推断为 Vec<usize>，而函数签名要求 Vec<i32>。需要用 as i32 显式转换。

错误 3：整数下溢

// ❌ 当 nums 为空时，len = 0, 0 - 1 在 usize 下 panic
let mut j = len - 1;

// ✅ 提前检查并返回
if len < 2 { return vec![]; }
let mut j = len - 1;

usize 是无符号类型，0 - 1 在 debug 模式下会 panic 溢出。

修正后的完整代码

impl Solution {
    pub fn two_sum(nums: Vec<i32>, target: i32) -> Vec<i32> {
        let len = nums.len();
        if len < 2 {
            return vec![];
        }
        let mut index_arr: Vec<usize> = (0..len).collect();
        index_arr.sort_by(|&a, &b| nums[a].cmp(&nums[b]));

        let mut i = 0;
        let mut j = len - 1;
        while i < j {
            match (nums[index_arr[i]] + nums[index_arr[j]]).cmp(&target) {
                std::cmp::Ordering::Less => i += 1,
                std::cmp::Ordering::Greater => j -= 1,
                std::cmp::Ordering::Equal => {
                    return vec![index_arr[i] as i32, index_arr[j] as i32];
                }
            }
        }
        vec![]
    }
}

涉及到的 Rust 知识点

#知识点说明
1impl 块与关联函数impl Solution { pub fn two_sum(...) }，没有 &self 是静态方法
2所有权与移动语义nums: Vec<i32> 值传参，所有权移入函数
3Range 与迭代器(0..len) 是 Range<usize>，实现了 Iterator
4类型推断(0..len).collect() 推断为 Vec<usize>
5闭包与排序sort_by(|a, b| ...) 捕获 nums，按值排序索引
6match 与 Ordering 枚举.cmp() 返回 Ordering::Less/Greater/Equal，必须穷尽
7usize 与索引数组索引必须是 usize，无符号类型有下溢风险
8as 类型转换usize as i32 显式转换
9vec! 宏vec![a, b] 创建 Vec，vec![] 创建空 Vec
10mut 可变绑定变量默认不可变，let mut i = 0 才能修改

简写技巧

// ✅ 自增简写
i += 1      // 等价于 i = i + 1

// ✅ 闭包参数解引用
sort_by(|&a, &b| ...)  // 相比于 |a, b| ... ，少写 *a, *b
这些知识点覆盖了 Rust 入门阶段最核心的领域。1、2、5、6、7 项必须完全理解，3、4、8 项可以先使用再逐步深入。
