LeetCode T8 · 字符串转换整数 atoi

这题可以看成一个小状态机：跳过前导空白、读取可选符号、连续读取数字，遇到第一个非法字符后停止。

状态与符号

当前解法用 sign 同时表示“是否已经进入解析状态”和“当前符号”。

sign含义
0还没有读到符号或数字
1当前结果按正数累积
-1当前结果按负数累积

遇到数字且还没有显式符号时，默认按正数处理：

sign = if sign == 0 { 1 } else { sign };

这样正数输入会让 result 一直保持正数，负数输入会让 result 一直保持负数，和整数反转题的思路一致。

溢出截断

helper 判断的是下一步 exist * 10 + last_digit 是否会越界：

fn add_with_clamp(exist: i32, last_digit: i32) -> i32 {
    if exist < i32::MIN / 10 || (exist == i32::MIN / 10 && last_digit < i32::MIN % 10) {
        return i32::MIN;
    }

    if exist > i32::MAX / 10 || (exist == i32::MAX / 10 && last_digit > i32::MAX % 10) {
        return i32::MAX;
    }

    exist * 10 + last_digit
}

边界里的最后一位分别是 7 和 -8，所以正数最多走到 2147483647，负数最多走到 -2147483648。

重要用例

输入结果原因
"42"42没有符号，默认正数
" -42"-42跳过前导空白后读取负号
"4193 with words"4193读到空格后停止
"words and 987"0第一个有效字符不是符号或数字
"+-12"0第二个符号出现时停止
"-91283472332"i32::MIN负数下溢时截断
"91283472332"i32::MAX正数上溢时截断

修正版

impl Solution {
    pub fn my_atoi(s: String) -> i32 {
        let bytes = s.trim_start().as_bytes();
        let mut result = 0i32;
        let mut sign = 0i32;

        for &b in bytes {
            let c = b as char;

            if !c.is_ascii_digit() && c != '+' && c != '-' {
                break;
            }

            if c == '+' || c == '-' {
                if sign != 0 {
                    break;
                }
                sign = if c == '+' { 1 } else { -1 };
                continue;
            }

            sign = if sign == 0 { 1 } else { sign };
            let digit = c.to_digit(10).unwrap() as i32;
            result = Solution::add_with_clamp(result, sign * digit);
        }

        result
    }

    fn add_with_clamp(exist: i32, last_digit: i32) -> i32 {
        if exist < i32::MIN / 10 || (exist == i32::MIN / 10 && last_digit < i32::MIN % 10) {
            return i32::MIN;
        }

        if exist > i32::MAX / 10 || (exist == i32::MAX / 10 && last_digit > i32::MAX % 10) {
            return i32::MAX;
        }

        exist * 10 + last_digit
    }
}

几个细节

  • trim_start() 比 trim() 更贴近题意：只跳过前导空白。
  • 符号只能在数字前出现一次；"+-12"、"-+12"、"12-3" 都会停止解析。
  • 前面已经判断 c.is_ascii_digit()，所以 to_digit(10).unwrap() 是安全的。
记忆点：累积过程中就保持正确符号，溢出处理也在累积前完成。
