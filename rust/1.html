📘 从真实工程学 Rust：ProcessManager 知识讲义

本讲义完全基于一份真实的 Rust 进程管理代码，目标是：

  • 你已经是工程师（C++ / Python）
  • 你不是在“学语法”，而是在理解 Rust 的工程哲学

知识点 1：模块系统与 use

1️⃣ 知识点讲解

Rust 的 use 用于显式声明依赖，没有隐式导入。

2️⃣ 使用场景 & 注意事项

  • 大型工程中避免“代码从哪来的不知道”
  • use 是可读性的第一层文档

3️⃣ 示例代码


use std::collections::HashMap;
use std::sync::{Arc, Mutex};

4️⃣ C++ 对比

  • C++：#include 可能引入大量隐式内容
  • Rust：你用什么，就写什么

知识点 2：条件编译 cfg

1️⃣ 讲解

#[cfg(unix)] / #[cfg(windows)] 在编译期选择代码。

2️⃣ 场景

  • 跨平台 OS API
  • 避免运行期 if 判断

3️⃣ 示例


#[cfg(unix)]
use platform::UnixProcessExt;

4️⃣ C++ 对比

  • C++：#ifdef 易造成代码碎片
  • Rust：作用域明确、类型安全

知识点 3：struct 与可见性

1️⃣ 讲解

struct 只描述数据结构，不负责行为。

2️⃣ 场景

  • 明确 API 边界
  • 避免“全 public struct”

3️⃣ 示例


pub struct ProcessHandle {
  pub info: ProcessInfo,
  child: Arc<Mutex<Child>>,
}

4️⃣ C++ 对比

  • C++：struct / class 行为混杂
  • Rust：数据与行为强制分离

知识点 4：impl 与方法

1️⃣ 讲解

impl 是行为定义区。

2️⃣ 场景

  • 清晰 API
  • 隐藏内部状态

3️⃣ 示例


impl ProcessHandle {
  pub fn pid(&self) -> Option<u32> {
    self.info.pid
  }
}

4️⃣ C++ 对比

  • C++：this 指针可变
  • Rust：&self / &mut self 强制区分

知识点 5：Option / Result

1️⃣ 讲解

Rust 没有 null，没有异常。

2️⃣ 场景

  • 所有失败路径必须处理

3️⃣ 示例


pub fn pid(&self) -> Option<u32>

4️⃣ C++ 对比

  • C++：nullptr / exception
  • Rust：失败是类型的一部分

知识点 6：Arc / Mutex / RwLock

1️⃣ 讲解

Rust 并发 = 显式共享 + 显式同步。

2️⃣ 场景

  • 多线程 / async 共享数据

3️⃣ 示例


Arc<RwLock<HashMap<String, Arc<RwLock<ProcessHandle>>>>>

4️⃣ C++ 对比

  • C++：shared_ptr + mutex（容易忘）
  • Rust：不加锁编译不过

知识点 7：async / await（Tokio）

1️⃣ 讲解

async 是状态机，不是线程。

2️⃣ 场景

  • 高并发 IO

3️⃣ 示例


pub async fn spawn(&self, config: ProcessConfig)

4️⃣ C++ 对比

  • C++20：coroutine（复杂）
  • Rust：语言级 async

知识点 8：tokio::spawn 与 JoinHandle

1️⃣ 讲解

spawn 创建异步任务。

2️⃣ 场景

  • 后台监控

3️⃣ 示例


let task = tokio::spawn(async move { ... });

4️⃣ C++ 对比

  • C++：std::thread / future
  • Rust：生命周期受控

知识点 9：测试（#[tokio::test]）

1️⃣ 讲解

Rust 把测试当一等公民。

2️⃣ 场景

  • 真实进程测试

3️⃣ 示例


#[tokio::test]
async fn test_spawn_process() { }

4️⃣ C++ 对比

  • C++：测试框架分裂
  • Rust：官方集成
