覆盖 Rust 核心智能指针：Box、Rc、Arc、Cell、RefCell、Mutex、MutexGuard、Weak。

1选型速查表

需求工具线程安全
唯一所有权 + 堆分配Box<T>✅（移动语义）
多所有者 + 不可变Rc<T>❌ 单线程
多所有者 + 不可变Arc<T>✅ 多线程
内部可变（Copy 类型）Cell<T>❌ 单线程
内部可变（运行时借用检查）RefCell<T>❌ 单线程
多线程互斥访问Mutex<T>✅
多线程多读单写RwLock<T>✅
弱引用（打破循环）Weak<T>跟随 Rc/Arc

2常用组合模式

单线程共享不可变    →  Rc<T>
单线程共享可变      →  Rc<RefCell<T>>
单线程内部可变      →  Cell<T> / RefCell<T>
多线程共享不可变    →  Arc<T>
多线程共享可变      →  Arc<Mutex<T>>
多线程多读单写      →  Arc<RwLock<T>>

3Box<T> — 堆分配

三个典型场景：

  • 递归类型 — enum List { Cons(i32, Box<List>), Nil }
  • 转移所有权避免拷贝 — 大数据结构
  • trait object — Box<dyn Trait>
let b = Box::new(42);   // 42 在堆上，b 在栈上
*b = 100;                // 通过 DerefMut 修改
// b 离开作用域 → Drop → 堆内存被释放

C++ 对应：std::unique_ptr<T>

4Rc<T> / Arc<T> — 引用计数

特性Rc<T>Arc<T>
引用计数方式非原子，快原子，线程安全
跨线程❌ 不是 Send✅ 是 Send + Sync
数据可变性只提供不可变引用只提供不可变引用
// Rc — 单线程共享
use std::rc::Rc;
let a = Rc::new(5);        // 计数 = 1
let b = a.clone();          // 计数 = 2（不拷贝数据）
println!("{}", Rc::strong_count(&a)); // 2

// Arc — 多线程共享
use std::sync::Arc;
let a = Arc::new(42);
let a1 = a.clone();
thread::spawn(move || { println!("{}", a1); });

C++ 对应：std::shared_ptr<T>

5Cell<T> / RefCell<T> — 内部可变性

特性Cell<T>RefCell<T>
取值方式.get() 拷贝值.borrow() 返回引用
设值方式.set(val) 替换值.borrow_mut() 返回可变引用
类型限制T 必须 Copy无限制
运行时开销零借用计数
// Cell — 简单值类型
use std::cell::Cell;
let c = Cell::new(42);
c.set(100);              // 不用 &mut self
println!("{}", c.get()); // 100

// RefCell — 复杂类型
use std::cell::RefCell;
let data = RefCell::new(vec![1, 2, 3]);
data.borrow_mut().push(4);   // 运行时检查借用规则
println!("{:?}", data.borrow());

// ❌ 运行时 panic：可变借用未释放就尝试不可变借用
let r1 = data.borrow_mut();
let r2 = data.borrow();      // panic

6Mutex<T> / MutexGuard — 线程互斥

Mutex 不是指针，不实现 Deref。你需要 .lock() 拿到 MutexGuard，MutexGuard 才实现 Deref 和 DerefMut。

use std::sync::{Arc, Mutex};

let data = Arc::new(Mutex::new(vec![1, 2, 3]));
{
    let guard = data.lock().unwrap();  // MutexGuard<Vec<i32>>
    guard.push(4);                     // DerefMut → &mut Vec → .push()
}  // guard 被 drop → 自动解锁

println!("{:?}", data.lock().unwrap()); // [1, 2, 3, 4]

Rust 的 Mutex 把数据包在锁里面 — 你不可能拿到没加锁的数据引用。C++ 的 std::mutex 和数据是分离的，忘加锁编译器不报错。

C++ 对比

// C++ — 锁和数据分离
std::mutex m;
std::vector<int> v;       // 数据外面的，没有编译期保护
{ std::lock_guard lk(m); v.push_back(4); }

7Weak<T> — 打破循环引用

use std::rc::{Rc, Weak};
use std::cell::RefCell;

struct Node {
    parent: RefCell<Weak<Node>>,        // 弱引用 — 不增加计数
    children: RefCell<Vec<Rc<Node>>>,   // 强引用 — 子节点活着，父才不被释放
}

// 父 → 子：Rc（强引用）
// 子 → 父：Weak（弱引用，子被释放父不受影响）

两个 Rc 互相指着会导致引用计数永久不为 0，内存泄漏。用 Weak 打断其中一条方向。

8MutexGuard 为什么能调 Vec 的方法

guard.push(4);  // guard 是 MutexGuard<Vec<i32>>，Vec 的 .push() 怎么能调？

因为标准库给 MutexGuard<T> 实现了 DerefMut<Target=T>，编译器自动插入解引用：

guard.push(4)
  → MutexGuard 没有 .push()
  → &MutexGuard 没有 .push()
  → &mut MutexGuard 没有 .push()
  → DerefMut → *T = Vec<i32>
  → &mut Vec 有 .push() ✅
MutexGuard 是一个"带自动解锁功能的包装器"。Deref 让你无感操作内部数据，Drop 在离开作用域时自动释放锁。
