String、str 与字符串方法完整心智模型

这篇笔记整理 Rust 字符串里最容易混淆的一组概念：str、&str、String、&&str，以及常见方法 to_string()、repeat()、as_str()、as_bytes()、bytes()、chars()。

1. 三个核心类型

类型一句话C/C++ 类比常见用法
str字符串内容本体，不定长，通常不能裸用。不定长 char[] / UTF-8 字节序列的语义版本。几乎总是藏在 &str、Box<str> 后面。
&str对一段字符串内容的借用视图。更像 std::string_view，不是简单的 char*。函数参数只读字符串时优先用它。
String拥有所有权、可增长、可修改的堆字符串。std::string。需要构造、修改、保存、返回字符串时用它。
记忆：String 是拥有者，&str 是观察窗口，str 是窗口背后的字符串内容本体。

2. 为什么不能直接写裸 str

str 是动态大小类型。"hi" 是 2 字节，"hello" 是 5 字节，"你好" 是 6 字节。编译器无法给裸 str 在栈上分配固定大小。

// 错误：裸 str 不知道大小
// let s: str = "hello";

// 正确：用 &str 借用一段字符串内容
let s: &str = "hello";

&str 是一个胖指针，包含两部分：

&str
┌──────────────┐
│ ptr ─────────┼──> UTF-8 字节起点
│ len          │
└──────────────┘

所以 &str 不依赖 \0 结尾，且保证内容是合法 UTF-8。

3. 字符串字面量是什么

let s = "hello";

这里 s 的类型是：

&'static str

意思是：它借用程序静态只读区域里的一段字符串数据，生命周期贯穿整个程序。

let borrowed: &str = "hello";
let owned: String = String::from("hello");
let owned2: String = "hello".to_string();

4. String 和 &str 如何互转

目标写法是否分配新内存说明
&str -> String"abc".to_string()是复制字符串内容，生成拥有者。
&str -> StringString::from("abc")是和 to_string() 在这里效果接近。
String -> &strs.as_str()否借用 String 的内容。
String -> &str&s否依靠 deref coercion，从 &String 转成 &str。
String -> &[u8]s.as_bytes()否借用底层 UTF-8 字节。
fn read_only(text: &str) {
    println!("{}", text);
}

let a = "hello";
let b = String::from("world");

read_only(a);       // &str
read_only(&b);      // &String 自动转成 &str
read_only(b.as_str());
函数只是读取字符串时，参数优先写 &str。这样既能接收字符串字面量，也能接收 String 的借用。

5. &&str 是什么

&&str 是“对 &str 的引用”。先有一个字符串切片变量：

let s: &str = "M";
let r: &&str = &s;

let a: &str = s;
let b: &str = *r; // 解一层引用

图上看是这样：

r: &&str
   │
   ▼
s: &str
   │ ptr + len
   ▼
"M"

为什么 HashMap<i32, &str>::get 返回 &&str

HashMap::get 的签名可以简化成：

fn get(&self, key: &K) -> Option<&V>

如果表是：

let table = HashMap::from([
    (1000, "M"),
    (900, "CM"),
]);

那么 value 类型是 &str，也就是：

V = &str
&V = &&str

所以：

if let Some(roman) = table.get(&1000) {
    // roman: &&str
    result.push_str(roman);  // 自动解引用成 &str
    result.push_str(*roman); // 显式写法
}

6. 对临时值取引用：table.get(&(digit * base))

这句代码的含义是：先算出临时 key，再临时借用它一下，交给 get 查表。

table.get(&(digit * base))

等价于：

let key = digit * base;
table.get(&key);

临时值的引用只需要活到这次函数调用结束。get 不会保存这个 key 引用，它只用它计算哈希和比较 key。返回值指向的是 HashMap 内部保存的 value，不是这个临时 key。

这就是为什么 some_map.get(&42) 是合法的：42 也是一个临时值，借用它只用于本次查找。

repeat() 的返回值为什么是 String

let x = "I".repeat(3);
assert_eq!(x, "III");

重复后的内容是新构造出来的字符串，必须有人拥有它，所以返回 String。

as_str() 和 to_string() 是相反方向

let owned: String = "hello".to_string(); // &str -> String，复制内容
let view: &str = owned.as_str();         // String -> &str，只借用

to_string() 会创建拥有者；as_str() 只是拿一个视图。

8. bytes()、as_bytes()、chars() 的选择

场景推荐原因
只处理 ASCII，例如 LeetCode 8 的数字字符as_bytes() 或 bytes()按字节处理更直接，速度快。
需要处理中文、emoji、非 ASCII 字符chars() / char_indices()不会把一个字符拆成多个字节。
既要字符，又要下标char_indices()返回的是字节下标，不是第几个字符。
循环里按下标取第 i 个字符避免 chars().nth(i)nth(i) 每次从头走，容易 O(n²)。
let s = "a你";

assert_eq!(s.as_bytes().len(), 4); // a: 1 字节，你: 3 字节
assert_eq!(s.chars().count(), 2);  // 两个 Unicode 标量值

9. 一套实用判断规则

  • 要返回新拼出来的字符串：用 String。
  • 函数只读字符串参数：写 &str。
  • 看到 &&str：通常是“容器返回 value 的引用，而 value 本身又是引用”。
  • 想从 String 暂时看一眼内容：用 as_str() 或 &s。
  • 想从 &str 变成可拥有、可修改的字符串：用 to_string() 或 String::from()。
  • ASCII 算法可以用字节；通用字符串算法优先用字符迭代。
