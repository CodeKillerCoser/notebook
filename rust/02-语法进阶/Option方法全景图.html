Option 方法全景图

速查表6 大类 30+ 方法

一、检查类（不消费，返回 bool）

方法签名简述说明
is_some()&self → bool是 Some 返回 true
is_none()&self → bool是 None 返回 true
is_some_and(f)&self, f → boolSome 且 f 返回 true（1.70+）

二、取出类（消费原值，拿走所有权）

方法签名简述None 时的行为
unwrap()self → Tpanic
expect(msg)self → Tpanic（带自定义消息）
unwrap_or(default)self → T返回 default
unwrap_or_else(f)self → T调用 f() 生成默认值
unwrap_or_default()self → T返回 T::default()
⚠️ 全部消费原值。调用后原变量无法再使用。

三、借用类（不消费，只是看一眼）

方法输入返回说明
as_ref()&Option<T>Option<&T>借出只读引用
as_mut()&mut Option<T>Option<&mut T>借出可变引用
as_deref()&Option<T>Option<&T::Target>借出引用并自动解引用一层
as_deref_mut()&mut Option<T>Option<&mut T::Target>同上，可变版

as_deref 使用场景

当 T 本身是 Box<U> 这种实现了 Deref 的类型时，能一步到位拿到最内层引用：

p: Option<Box<ListNode>>
p.as_ref()   → Option<&Box<ListNode>>   // 还得再 .val 或 ** 解引用
p.as_deref() → Option<&ListNode>        // 直接拿到 &ListNode，可以直接 .val

四、转换/映射类

方法消费？说明
map(f)✔️ 消费Some 时用 f 转换内容，自动包成 Some
map_or(default, f)❌ 不消费Some 时 f 处理，否则返回 default
map_or_else(default_fn, f)❌ 不消费同上，default 由函数生成
and_then(f)✔️ 消费f 返回 Option，类似 flatMap
inspect(f)❌ 不消费执行副作用（打日志等），返回自身（1.76+）

map vs and_then

// map: f 返回普通值，自动包成 Some
Some(2).map(|x| x * 2)        // → Some(4)
None.map(|x| x * 2)           // → None

// and_then: f 返回 Option，用于链式调用
p.and_then(|node| node.next)  // → p 的 next，p 是 None 则直接返回 None

五、备用/替换类

方法说明
or(alt)None 时用 alt 替换（消费 alt）
or_else(f)None 时调用 f() 生成替换值
replace(&mut self, value: T) → Option<T>用 value 替换原值，返回旧值
take(&mut self) → Option<T>取出值，原位置变 None
insert(&mut self, value: T) → &mut T插入值，返回内部可变引用
p.or(q)        // p 有值用 p，否则用 q（注意 q 也被消费）
p.or_else(|| Some(Box::new(ListNode::new(0))))  // 懒加载备胎

六、过滤/组合/转换类

方法说明
filter(f)Some 时如果 f 返回 false 则变成 None
flatten()Option<Option<T>> → Option<T>
zip(other)两个都是 Some → Some((a, b))，否则 None
zip_with(other, f)两个都是 Some → Some(f(a, b))
ok_or(err)Option<T> → Result<T, E>
ok_or_else(f)同上，err 由函数生成
transpose()Option<Result<T,E>> ↔ Result<Option<T>,E>
xor(other)恰好一个为 Some 时返回那个 Some
get_or_insert(default)None 时插入 default，返回 &mut T
get_or_insert_with(f)None 时插入 f() 结果，返回 &mut T
cloned()Option<&T> → Option<T>（T: Clone）
copied()Option<&T> → Option<T>（T: Copy）

记忆技巧：快递包裹类比

操作方法类比

隔着包装看标签as_ref()从透明窗口看里面，包装完好
拆开，空的就报警expect("...")空的就喊"你给我个空包裹？！"
拆开，空的就用备用的unwrap_or(x)空的就用自己的
拆开，换货再包上map(f)换货后重新封装

检查后退回filter(f)不合格就退回
