Option 常用组合模式

实战模式4 个高频场景

1 借读取值

场景：我有一个 Option<Box<ListNode>>，需要读 val，但不想消费它。

// p: Option<Box<ListNode>>，我要读 val，但不消费 p
let x = p.as_ref().map_or(0, |node| node.val);
//      ^^^^^^^^ 借出引用
//               ^^^^^^^ 有值就处理，无值给默认

原理：as_ref() 把 Option<Box<T>> 变成 Option<&Box<T>>，不拿走所有权。map_or 的签名是 &self，同样不消费。

2 链式消费

场景：链表遍历中，消费当前节点，取出 next 赋给自己。

// 消费 p，取出 next 赋值给 p（典型链表遍历）
p = p.and_then(|node| node.next);
//  ^^^^^^^^ node.next 是 Option<Box<ListNode>>，正好是 p 的类型

原理：and_then 接收 FnOnce(T) → Option<U>。如果 p 是 None，返回 None；如果是 Some，取出内部值传给闭包，闭包返回 node.next。

3 零方法模式匹配

场景：对方法调用还不熟时，用 match 最安全直观。

let (x, next_p) = match p {
    Some(node) => (node.val, node.next),
    None => (0, None),
};

注意事项：

  • 如果不想消费 p，用 match &p（引用匹配）
  • 如果想消费 p，用 match p（值匹配）

4 可变借用修改内部值

场景：拿到链表尾节点的可变引用，修改其内部字段。

// tail: &mut Box<ListNode>
tail.next = Some(Box::new(ListNode::new(42)));  // 追加新节点

// 或修改已有节点
tail.next.as_mut().unwrap().val = 42;  // 修改链表节点的值

原理：as_mut() 把 &mut Option<T> 变成 Option<&mut T>，拿到内部的可变引用但不消费。

记忆法则：
读值 → as_ref + map_or
推进 → and_then
拆解 → match
修改 → as_mut + unwrap
