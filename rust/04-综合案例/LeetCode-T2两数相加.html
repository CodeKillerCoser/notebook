LeetCode T2 · 两数相加

链表所有权Option

题目：两个逆序存储的链表，每位相加，返回新链表。

1原始错误代码

pub fn add_two_numbers(
    l1: Option<Box<ListNode>>,
    l2: Option<Box<ListNode>>,
) -> Option<Box<ListNode>> {
    let mut carry = 0;
    let mut head = Box::new(ListNode::new(0));
    let mut tail = &mut head;
    let mut p = l1;
    let mut q = l2;

    while p.is_some() || q.is_some() {
        // ❌ 错误① & ②：unwrap() 消费了 p 和 q
        let x = if p.is_some() { p.unwrap().val } else { 0 };
        let y = if q.is_some() { q.unwrap().val } else { 0 };
        //    第一次循环后 p、q 已被 move

        let sum = x + y + carry;
        carry = sum / 10;

        tail.next = Some(Box::new(ListNode::new(sum % 10)));
        tail = tail.next.as_mut().unwrap();

        // ❌ 错误③ & ④：p、q 已被消费，再次访问报 "use of moved value"
        p = if p.is_some() { p.unwrap().next } else { None };
        q = if q.is_some() { q.unwrap().next } else { None };
    }

    // ❌ 错误⑤：tail 还在可变借用 head
    return head.next;
}

2错误清单

编号位置类型原因
①p.unwrap().valuse of moved valueunwrap() 消费 p，后面又用 p
②q.unwrap().valuse of moved value同上，q 被消费
③p.is_some() 第二处use of moved valuep 在第①处已被 move
④q.is_some() 第二处use of moved value同上
⑤return head.nextborrow conflicttail 还在可变借用 head
⑥逻辑 bug缺少进位处理循环后 carry > 0 时需追加节点

3修复版一（推荐 · 方法式）

用 as_ref 借读值、and_then 链式推进。

pub fn add_two_numbers(
    l1: Option<Box<ListNode>>,
    l2: Option<Box<ListNode>>,
) -> Option<Box<ListNode>> {
    let mut carry = 0;
    let mut head = Box::new(ListNode::new(0));
    let mut tail = &mut head;
    let mut p = l1;
    let mut q = l2;

    while p.is_some() || q.is_some() || carry > 0 {
        // ✅ 用 as_ref() 借用，不消费 p/q
        let x = p.as_ref().map_or(0, |node| node.val);
        let y = q.as_ref().map_or(0, |node| node.val);

        let sum = x + y + carry;
        carry = sum / 10;

        tail.next = Some(Box::new(ListNode::new(sum % 10)));
        tail = tail.next.as_mut().unwrap();

        // ✅ 用 and_then 消费当前 p/q 并取出 next
        p = p.and_then(|node| node.next);
        q = q.and_then(|node| node.next);
    }

    head.next  // ✅ tail 不再使用，可安全访问 head
}

4修复版二（零方法调用 · match）

对方法还不熟时的安全写法。

pub fn add_two_numbers(
    l1: Option<Box<ListNode>>,
    l2: Option<Box<ListNode>>,
) -> Option<Box<ListNode>> {
    let mut carry = 0;
    let mut head = Box::new(ListNode::new(0));
    let mut tail = &mut head;
    let mut p = l1;
    let mut q = l2;

    while p.is_some() || q.is_some() || carry > 0 {
        let x = match &p {
            Some(node) => node.val,
            None => 0,
        };
        let y = match &q {
            Some(node) => node.val,
            None => 0,
        };

        let sum = x + y + carry;
        carry = sum / 10;

        tail.next = Some(Box::new(ListNode::new(sum % 10)));
        tail = tail.next.as_mut().unwrap();

        p = match p {
            Some(node) => node.next,
            None => None,
        };
        q = match q {
            Some(node) => node.next,
            None => None,
        };
    }

    head.next
}

5核心领悟

  1. unwrap() 的 self 是值不是引用——调用后变量即废，签名里不带 & 的都消费所有权
  2. 循环中用 as_ref / match &p 是常态——循环变量必须复用
  3. and_then 是链表的天然搭档——"有就往下走，没有就停"
  4. 借用冲突靠作用域解决——变量不用了就自动释放借用
经验法则：遇到 Option 操作编译错误时，先检查是「消费后使用」还是「借用了又访问」——90% 的错误都在这两类里。
