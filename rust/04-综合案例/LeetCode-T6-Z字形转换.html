这题适合直接按行模拟。维护当前行 cur_row 和方向 going_down，到顶部或底部时切换方向。

1当前解法

impl Solution {
    pub fn convert(s: String, num_rows: i32) -> String {
        if s.is_empty() || num_rows <= 1 {
            return s;
        }

        let mut rows = vec![String::new(); num_rows as usize];
        let mut cur_row = 0;
        let mut going_down = false;
        for (_, c) in s.chars().enumerate() {
            rows[cur_row].push(c);
            if cur_row == 0 {
                going_down = true;
            } else if cur_row == num_rows as usize - 1 {
                going_down = false;
            }
            cur_row = if going_down { cur_row + 1 } else { cur_row - 1 };
        }

        rows.into_iter().fold(String::new(), |acc, row| acc + &row)
    }
}

2Review 结论

核心算法没有明显问题，常规 LeetCode 用例可以通过。复杂度为：

复杂度结果原因
时间O(n)每个字符处理一次。
空间O(n)每一行的临时字符串合计保存全部字符。

3可以优化的地方

  • enumerate() 的下标没有使用，可以直接写 for c in s.chars()。
  • num_rows as usize 在循环中重复转换，可以提前绑定。
  • 当 num_rows >= s.chars().count() 时，结果就是原字符串，可以提前返回。
  • fold(String::new(), |acc, row| acc + &row) 可读性一般，更推荐 rows.concat()。

4优化版

impl Solution {
    pub fn convert(s: String, num_rows: i32) -> String {
        let num_rows = num_rows as usize;

        if s.is_empty() || num_rows <= 1 || num_rows >= s.chars().count() {
            return s;
        }

        let mut rows = vec![String::new(); num_rows];
        let mut cur_row = 0usize;
        let mut going_down = false;

        for c in s.chars() {
            rows[cur_row].push(c);

            if cur_row == 0 {
                going_down = true;
            } else if cur_row == num_rows - 1 {
                going_down = false;
            }

            cur_row = if going_down { cur_row + 1 } else { cur_row - 1 };
        }

        rows.concat()
    }
}
记忆点：这题的重点不是复杂算法，而是把“当前在哪一行”和“接下来往哪走”两个状态维护清楚。
