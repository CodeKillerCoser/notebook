这题的核心是滑动窗口：窗口右端不断扩张，遇到重复字符时把左端移动到重复字符上一次出现位置之后。

1原始思路

pub fn length_of_longest_substring(s: String) -> i32 {
    let len = s.len();
    let mut index_map = std::collections::HashMap::new();
    let mut start = 0;
    let mut max_len = 0;
    for (i, c) in s.char_indices() {
        if let Some(pre) = index_map.get(&c) {
            max_len = max_len.max(i - start);
            start = start.max(pre + 1);
        }
        index_map.insert(c, i);
    }
    return max_len.max(len - start) as i32;
}

2关键问题

点说明
char_indices()返回的是字节下标，不是字符下标。纯 ASCII 场景没问题；遇到中文或 emoji 时长度会偏大。
max_len 更新时机只在遇到重复字符时更新，最后再补一次是可行的，但每轮更新更直观。
HashMap::getif let Some(&pre) 比 Some(pre) 再处理引用更清晰。

3LeetCode ASCII 优化版

这题约束通常是英文字母、数字、符号和空格，可以用数组代替 HashMap，常数性能更好。

impl Solution {
    pub fn length_of_longest_substring(s: String) -> i32 {
        let bytes = s.as_bytes();
        let mut last = [usize::MAX; 128];
        let mut start = 0usize;
        let mut ans = 0usize;

        for (i, &b) in bytes.iter().enumerate() {
            let idx = b as usize;

            if last[idx] != usize::MAX && last[idx] >= start {
                start = last[idx] + 1;
            }

            ans = ans.max(i - start + 1);
            last[idx] = i;
        }

        ans as i32
    }
}

4Unicode 友好版

use std::collections::HashMap;

impl Solution {
    pub fn length_of_longest_substring(s: String) -> i32 {
        let mut last = HashMap::new();
        let mut start = 0usize;
        let mut ans = 0usize;

        for (i, c) in s.chars().enumerate() {
            if let Some(&pre) = last.get(&c) {
                start = start.max(pre + 1);
            }

            ans = ans.max(i - start + 1);
            last.insert(c, i);
        }

        ans as i32
    }
}
记忆点：窗口左边界只前进不后退，所以更新时要用 start.max(pre + 1)。
