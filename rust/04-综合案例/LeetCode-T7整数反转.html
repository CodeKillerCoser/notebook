这题的关键是让累积结果始终保持原数字的符号，并在执行 reverse_num * 10 + last_digit 之前完成溢出判断。

1符号处理

Rust 的整数取余会保留被除数的符号：

123 % 10 == 3
-123 % 10 == -3

因此负数不需要先取绝对值，也不需要最后再乘回符号。每次取到的 last_digit 已经带着正确符号。

2溢出判断

i32 边界拆成“前 9 位”和“最后 1 位”来看：

边界除以 10最后一位
i32::MAX = 21474836472147483647
i32::MIN = -2147483648-214748364-8

当 reverse_num 已经到达边界的十分之一时，就必须比较最后一位。

if reverse_num > i32::MAX / 10
    || (reverse_num == i32::MAX / 10 && last_digit > i32::MAX % 10)
{
    return 0;
}

if reverse_num < i32::MIN / 10
    || (reverse_num == i32::MIN / 10 && last_digit < i32::MIN % 10)
{
    return 0;
}

3修正版

impl Solution {
    pub fn reverse(mut x: i32) -> i32 {
        let mut reverse_num = 0i32;

        while x != 0 {
            let last_digit = x % 10;

            if reverse_num > i32::MAX / 10
                || (reverse_num == i32::MAX / 10 && last_digit > i32::MAX % 10)
            {
                return 0;
            }

            if reverse_num < i32::MIN / 10
                || (reverse_num == i32::MIN / 10 && last_digit < i32::MIN % 10)
            {
                return 0;
            }

            reverse_num = reverse_num * 10 + last_digit;
            x /= 10;
        }

        reverse_num
    }
}

4草稿里的小问题

  • 多余的 }) 应删掉，只保留函数或 impl 需要的右花括号。
  • 已经保存了 let last_digit = x % 10;，后面直接复用即可，没必要再写一次 x % 10。
记忆点：不要等乘法和加法发生后再判断溢出，先判断下一步是否安全。
