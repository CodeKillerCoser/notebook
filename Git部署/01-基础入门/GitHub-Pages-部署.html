GitHub Pages 是一个免费的静态站点托管服务，直接从 GitHub 仓库中获取文件渲染网页。

1为什么访问链接只显示 README

GitHub Pages 的渲染优先级：

  1. 如果根目录有 index.html，渲染它
  2. 如果没有 index.html，回退渲染 README.md
  3. 不会 自动生成文件目录列表（不像 Apache/Nginx 有 Directory Listing）

所以当仓库只有 README.md 时，Pages 只展示那一段文字。

2正确部署方式（三种方案）

方案一：手动创建 index.html（最简单）

在根目录放一个 index.html，用文件导航列出所有笔记链接。适合文件不多的场景。


.
├── index.html        ← 导航页面，列出所有文件的链接
├── README.md
├── rust/
│   └── 1.html
└── 代码大全阅读笔记/
    ├── ch2-理解编程.md
    └── ch3-软件开发先决条件.md

方案二：Jekyll（GitHub Pages 原生支持）

根目录放 _config.yml 和 index.md，GitHub Pages 自动用 Jekyll 生成站点。但默认只处理 _posts 目录，非标准笔记结构需额外配置。

方案三：静态站点生成器 + GitHub Actions

用 MkDocs、docsify 等工具，通过 GitHub Actions 部署到 gh-pages 分支，每次 push 自动重新构建发布。

3GitHub Pages 设置检查

在仓库的 Settings → Pages 中确认：

1. Source: Deploy from a branch
2. Branch: main, 目录选 / (root)
3. 保存

4关键结论

GitHub Pages 只是静态文件服务器，不会自动生成目录列表。要使站点有导航，需要提供一个 index.html。
