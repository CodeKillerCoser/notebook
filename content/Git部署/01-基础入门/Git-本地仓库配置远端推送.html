Git 本地仓库配置远端推送

在本地新建 Git 仓库后，即使已经能正常 git commit，也不代表可以直接 git push。推送需要先配置远端仓库。

现象

git push
fatal: No configured push destination.
Either specify the URL from the command-line or configure a remote repository using

    git remote add <name> <url>

and then push using the remote name

    git push <name>

原因

当前分支已经有本地 commit，但仓库没有 origin 或其他 remote，因此 Git 不知道要把提交推送到哪里。

检查方式

git status --short
git branch --show-current
git remote -v

如果 git remote -v 没有任何输出，说明还没有配置远端。

解决方式

git remote add origin <你的仓库地址>
git push -u origin master

如果远端默认分支是 main，则推送到 main：

git branch -M main
git push -u origin main
记忆点：commit 只发生在本地仓库，push 需要远端地址和上游分支。
