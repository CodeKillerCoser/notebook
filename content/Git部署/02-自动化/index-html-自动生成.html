GitHub Actions 自动生成 index.html

通过 GitHub Actions 在每次 push 时自动扫描目录结构、生成 index.html 并推送回仓库，无需手动维护文件浏览器。

整体流程


push → GitHub Actions(ubuntu) → python generate_index.py → git commit → git push → Pages 自动更新

核心文件结构


notebook/
├── generate_index.py              ← 扫描目录、生成 index.html 的 Python 脚本
├── .github/workflows/
│   └── generate-index.yml         ← GitHub Actions workflow
└── index.html                     ← 自动生成的结果

generate_index.py 工作逻辑

目录扫描

  • 递归遍历仓库所有文件和目录
  • 排除 .git、.deepseek、__pycache__、node_modules
  • 排除 generate_index.py、index.html、README.md 自身
  • 目录排在文件前面，按名称排序

HTML 生成特性

  • 可折叠目录 — 点击 ▶ 展开/收起子目录
  • 文件图标 — 按扩展名区分（.md 蓝、.html 红、.rs 紫等）
  • 深色模式 — 跟随系统 prefers-color-scheme
  • README 快捷链接 — 顶部单独展示，树中不重复
  • 自动生成时间戳 — 页脚标注生成时间

GitHub Actions Workflow

name: Generate index.html

on:
  push:
    branches: [main]    # 每次 push 到 main 触发
  workflow_dispatch:    # 也支持手动触发

permissions:
  contents: write       # 需要写权限才能 push 回仓库

jobs:
  generate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: python generate_index.py
      - run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add index.html
          if git diff --staged --exit-code; then
            echo "No changes"
          else
            git commit -m "auto: regenerate index.html"
            git push
          fi

关键点

  • Workflow 使用 contents: write 权限，需要 GitHub 授予
  • commit 前检查 git diff --staged --exit-code，无变化不提交，避免产生空 commit
  • GitHub Pages 配置 Source 为 main / / (root)
每次添加新笔记文件 → push 到 main → 自动重新生成 index.html → 无需手动维护。
