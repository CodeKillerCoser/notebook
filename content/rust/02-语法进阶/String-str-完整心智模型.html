1. 三个核心类型：str、&str、String

类型一句话C/C++ 类比常见用法
str字符串内容本体，不定长，通常不能裸用。不定长 char[] / UTF-8 字节序列的语义版本。几乎总是藏在 &str、Box<str>、Rc<str> 后面。
&str对一段字符串内容的借用视图。更像 std::string_view，不是简单的 char*。函数参数只读字符串时优先用它。
String拥有所有权、可增长、可修改的堆字符串。std::string。需要构造、修改、保存、返回字符串时用它。
记忆：String 是拥有者，&str 是观察窗口，str 是窗口背后的字符串内容本体。

2. 内存模型：为什么 str 不能裸用

str 是动态大小类型。"hi" 是 2 字节，"hello" 是 5 字节，"你好" 是 6 字节。编译器无法给裸 str 在栈上分配固定大小。

// 错误：裸 str 不知道大小
// let s: str = "hello";

// 正确：用 &str 借用一段字符串内容
let s: &str = "hello";

&str 是胖指针，包含指针和长度：

&str
┌──────────────┐
│ ptr ─────────┼──> UTF-8 字节起点
│ len          │
└──────────────┘

String 可以近似理解为三元组：

String
┌──────────────┐
│ ptr ─────────┼──> 堆上的 UTF-8 字节缓冲区
│ len          │
│ capacity     │
└──────────────┘

&str 不依赖 \0 结尾，且保证内容是合法 UTF-8。C 的 const char* 只有指针，长度要靠 \0 推断；Rust 的 &str 自带长度，更接近 C++ 的 std::string_view。

3. 字符串字面量与生命周期

let s = "hello";

这里 s 的类型是 &'static str：它借用程序静态只读区域里的一段字符串数据，生命周期贯穿整个程序。

let borrowed: &str = "hello";
let owned: String = String::from("hello");
let owned2: String = "hello".to_string();

字符串字面量可以理解成“不可变的静态字符串内容 + 一个 &str 视图”。它不是 String，也不是裸 str。

4. 创建与转换

目标写法是否分配说明

&str -> StringString::from("abc")是和 to_string() 在这里效果接近。
创建空字符串String::new()否或延迟常用于逐步拼接。
预分配容量String::with_capacity(n)是已知大概长度时减少扩容。

String -> &str&s否依靠 deref coercion，从 &String 转成 &str。
String -> Vec<u8>s.into_bytes()否消费 String，拿到底层字节缓冲区。
Vec<u8> -> StringString::from_utf8(bytes)通常否检查 UTF-8 合法性，成功后复用缓冲区。

fn read_only(text: &str) {
    println!("{}", text);
}

let a = "hello";
let b = String::from("world");

read_only(a);          // &str
read_only(&b);         // &String 自动转成 &str
read_only(b.as_str()); // 显式借用为 &str
函数只是读取字符串时，参数优先写 &str。这样既能接收字符串字面量，也能接收 String 的借用。

5. 所有权与借用：什么时候用 String，什么时候用 &str

场景推荐类型原因
函数只读，不保存&str最灵活，接受字面量和 String。
函数要返回新构造的字符串String返回值必须拥有数据。
结构体字段要长期保存String结构体通常需要拥有字段数据。
结构体字段只是借用外部数据&'a str需要生命周期参数，复杂度更高。
错误类型、命令参数、路径等非 UTF-8 友好场景OsString / OsStr文件系统路径不一定是合法 UTF-8。
C FFI 以 \0 结尾的字符串CString / CStr和 C 字符串约定匹配。

6. &&str、临时引用与容器返回值

&&str 是“对 &str 的引用”。如果容器的 value 本身是 &str，而 get 又返回 value 的引用，就会得到 &&str。

use std::collections::HashMap;

let table = HashMap::from([
    (1000, "M"),
    (900, "CM"),
]);

if let Some(roman) = table.get(&1000) {
    // roman: &&str
    result.push_str(roman);  // 自动解引用成 &str
    result.push_str(*roman); // 显式写法
}

HashMap::get 的签名可以简化成：

fn get(&self, key: &K) -> Option<&V>

如果 V = &str，那么 &V = &&str。

这句代码：

table.get(&(digit * base))

等价于：

let key = digit * base;
table.get(&key);

临时值的引用只需要活到这次函数调用结束。get 不保存这个 key 引用，它只用它计算哈希和比较 key。返回值指向的是 HashMap 内部保存的 value，不是这个临时 key。

7. 字节、字符、Unicode 与下标

Rust 字符串是 UTF-8。一个字符不一定等于一个字节。

let s = "a你";

assert_eq!(s.len(), 4);          // 字节数：a 1 字节，你 3 字节
assert_eq!(s.chars().count(), 2); // Unicode 标量值数量

Rust 不允许直接 s[0]，因为字符串按字节存储，直接按整数下标无法保证落在合法字符边界上。

let s = "a你";
assert!(s.is_char_boundary(0));
assert!(s.is_char_boundary(1));
assert!(!s.is_char_boundary(2)); // 落在“你”的中间字节
assert!(!s.is_char_boundary(3));
assert!(s.is_char_boundary(4));
let s = "a你";
let a = &s[0..1];  // OK
let ni = &s[1..4]; // OK
// let broken = &s[1..2]; // panic：不是字符边界
len() 是字节长度，不是字符数量。需要字符数量时用 chars().count()，但它必须遍历整个字符串。

8. 遍历方式总表

方法返回值视角适用场景是否分配
as_bytes()&[u8]借用整段 UTF-8 字节需要随机访问字节、只处理 ASCII。否
bytes()Iterator<Item = u8>逐字节顺序扫描 ASCII 或原始 UTF-8 字节。否
chars()Iterator<Item = char>逐 Unicode 标量值按字符处理中文、emoji、非 ASCII。否
char_indices()Iterator<Item = (usize, char)>字节下标 + 字符既需要字符，又需要能切片的字节下标。否
lines()Iterator<Item = &str>按行借用处理多行文本。否
split_whitespace()Iterator<Item = &str>按 Unicode 空白切分解析单词、输入参数。否
split(pattern)Iterator<Item = &str>按模式切分逗号、冒号、字符或闭包切分。否
match_indices(pattern)Iterator<Item = (usize, &str)>匹配位置 + 匹配文本需要定位所有匹配。否

不要在循环里频繁用 chars().nth(i)

// 错误倾向：nth(i) 每次从头走，整体容易 O(n^2)
for i in 0..s.chars().count() {
    let ch = s.chars().nth(i);
}

// 推荐：一次性迭代
for (i, ch) in s.char_indices() {
    println!("{} {}", i, ch);
}

9. 修改与拼接

方法/写法作用注意点
push(char)追加一个字符。参数是 char，不是 &str。
push_str(&str)追加字符串切片。不拿走参数所有权。
insert(idx, char)在字节下标处插入字符。idx 必须是字符边界。
insert_str(idx, &str)在字节下标处插入字符串。idx 必须是字符边界。
pop()移除并返回最后一个字符。返回 Option<char>。
remove(idx)移除并返回指定字节下标处的字符。idx 必须是字符边界，后续内容会移动。
truncate(idx)截断到指定字节长度。idx 必须是字符边界。
clear()清空内容。保留容量。
replace_range(range, &str)替换指定范围。范围必须落在字符边界上。
reserve(n)预留额外容量。减少多次拼接时的扩容次数。
shrink_to_fit()尽量收缩容量。是否完全收缩由分配器决定。
let mut s = String::with_capacity(16);
s.push('R');
s.push_str("ust");
s.insert_str(0, "Hello ");
assert_eq!(s, "Hello Rust");

+、format!、push_str() 的区别

let a = String::from("hello");
let b = String::from(" world");

let c = a + &b; // a 被移动，b 被借用
// println!("{}", a); // a 已不可用

let d = format!("{}{}", c, b); // 不移动 c 和 b，但会新建 String

let mut e = String::new();
e.push_str(&d); // 原地追加，适合循环构造

+ 背后类似 add(self, rhs: &str)，左边的 String 会被消费，右边通常借用为 &str。

10. 查找、判断、切分与清理

类别方法返回值/说明
判断is_empty()、starts_with()、ends_with()、contains()返回 bool。
查找find()、rfind()返回 Option<usize>，下标是字节下标。
去前后空白trim()、trim_start()、trim_end()返回 &str，不分配。
去前后指定字符trim_matches()、trim_start_matches()、trim_end_matches()返回 &str。
去前后缀strip_prefix()、strip_suffix()返回 Option<&str>。
切分split()、rsplit()、split_once()、splitn()多数返回借用切片的迭代器。
替换replace()、replacen()返回新的 String，会分配。
大小写to_lowercase()、to_uppercase()返回新的 String，Unicode 下长度可能变化。
let line = "  name=Rust  ";
let line = line.trim();                    // &str，不分配
let (key, value) = line.split_once('=').unwrap();

assert_eq!(key, "name");
assert_eq!(value, "Rust");

11. 解析、格式化与展示

需求写法说明
字符串转数字"42".parse::<i32>()返回 Result<i32, ParseIntError>。
数字转字符串42.to_string()通过 Display 生成 String。
拼格式化文本format!("x = {}", x)返回 String。

assert_eq!(text, "n = 42");

12. 方法速查：按是否分配分类

不分配，返回借用/迭代器会创建新字符串或新缓冲区
as_str()、as_bytes()、bytes()、chars()、char_indices()、lines()、split()、split_whitespace()、trim()、strip_prefix()、find()、contains() to_string()、String::from()、format!()、repeat()、replace()、replacen()、to_lowercase()、to_uppercase()

push()、push_str()、insert()、insert_str() 是原地修改 String；如果容量不足，可能触发重新分配。

13. 常见坑清单

坑原因建议
把 len() 当字符数len() 是 UTF-8 字节数。字符数用 chars().count()。
对中文字符串随便切片切片范围必须是字符边界。用 char_indices() 拿合法边界。
循环里用 chars().nth(i)每次从头遍历，容易 O(n^2)。一次性 chars() 或 char_indices()。
函数参数写 String会要求调用方交出所有权或 clone。只读参数写 &str。
看到 &&str 就慌value 本身是引用，容器又返回 value 的引用。需要时用 *x 解一层，很多 API 会自动解引用。
误以为 trim() 会修改原字符串trim() 返回借用切片。需要拥有结果时再 .to_string()。
误用 as_bytes() 处理 Unicode 字符多字节字符会被拆开。通用文本处理用 chars()。
用 + 后继续使用左操作数+ 会消费左边的 String。改用 format! 或 push_str()。

14. 实用决策树

你要处理字符串？
  │
  ├─ 只是读函数参数？
  │    └─ 用 &str
  │
  ├─ 要保存到结构体、返回新结果、持续拥有？
  │    └─ 用 String
  │
  ├─ 要遍历？
  │    ├─ 只处理 ASCII / 原始字节 → as_bytes() / bytes()
  │    ├─ 通用 Unicode 字符 → chars()
  │    └─ 字符 + 可切片下标 → char_indices()
  │
  ├─ 要切分/去空白/找前后缀？
  │    └─ 优先用返回 &str 的 split/trim/strip 系列，不急着 to_string()
  │
  ├─ 要拼接很多次？
  │    └─ String::with_capacity() + push_str()
  │
  └─ 要生成格式化文本？
       └─ format!()
