# 外部 HTML 笔记导入规则

## 移动端响应式检查

导入外部 HTML 或报告页时，不能只调整站点外层容器宽度。很多报告自带内部组件样式，例如 hero、对照图、表格、dashboard mock、grid/card 组合。这些内部样式可能仍按桌面端宽度渲染，导致移动端截图表现为内容被截断或页面出现横向滚动。

导入后必须检查：

- 外层页面在 390px 宽度下是否无横向滚动。
- 导入内容的根容器是否设置 `max-width: 100%` 和必要的 `overflow-x` 处理。
- 内部 grid 是否在移动端改为 `minmax(0, 1fr)` 或单列。
- 长标题是否重设移动端字号、取消负字距，并允许 `overflow-wrap: anywhere`。
- 表格、代码块、图形和 mock UI 是否要局部横向滚动，而不是撑宽整个页面。

经验结论：移动端问题不能只看站点 layout。外部文章若包含自己的设计系统或 demo 组件，必须同时审查文章内部 CSS 的断点。
