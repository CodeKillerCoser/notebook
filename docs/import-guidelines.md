# 文章内容与导入规则

博客模板统一负责导航、面包屑、标题、日期、标签、左侧目录、主题、字体和响应式布局。文章文件只提供元数据与语义正文，不定义页面样式。

通过对话或外部文件生成文章时使用 `.skills/kb-note-generator/SKILL.md`；修改模板、主题、字体、关键词或部署系统时使用 `.skills/notebook-site-maintainer/SKILL.md`。

## 推荐结构

```html
---
title: "文章标题"
date: "2026-07-13T00:00:00+08:00"
category: "主题"
tags:
  - "标签一"
  - "标签二"
---

<p>正文开场。</p>
<h2>一级章节</h2>
<p>章节内容。</p>
<h3>二级章节</h3>
<pre><code class="language-rust">fn main() {}</code></pre>
```

## 内容约束

- 不写 `<style>`、外部样式表或页面级内联样式。
- 不写 `<header>`、Hero、面包屑、标签栏、文章标题或自定义目录。
- 正文从段落或 `h2` 开始；模板是页面中唯一的 `h1` 来源。
- 使用 `h2` / `h3` 表达章节层级，左侧目录由构建器自动生成。
- 图片、表格、代码、引用、列表和语义化 callout 可以保留。
- 不使用 `.container`、`.layout`、`.page`、`.card` 等页面布局 class。

## 旧 HTML 导入

旧文章可以直接放入 `content/`。构建阶段会自动：

- 选取 `article` 或 `main` 中的正文内容并丢弃页面外壳。
- 删除脚本、样式、旧 Header、旧目录和旧面包屑。
- 删除重复标题；正文中其他 `h1` 会转换为 `h2`。
- 移除头部自定义标签行、内联页面样式和卡片 class。
- 为 `h2` / `h3` 生成稳定 ID，并为 `h2` 生成左侧目录。
- 统一代码高亮、主题颜色、字体与移动端行为。

导入后仍需运行 `npm run build` 和 `npm run check:links`，并抽查桌面与移动端页面。
