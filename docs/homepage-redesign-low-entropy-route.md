# 低熵航线首页改造方案

本文档整理“低熵航线”个人笔记博客的首页改造方向，供本地执行页面修改时参考。目标是借鉴 InfoQ 首页导航的清晰分区和内容入口意识，但保留个人博客的克制、轻量与长期笔记属性。

## 当前实施基线（2026-07-13）

后续维护以下列已落地决策为准；本文后续章节保留早期设计推演：

- 导航工具区包含搜索、RSS、外观和字体；主题与阅读字体均可持久化切换。
- 导航、Hero 与正文关键词星图合并在同一个全宽宇宙背景中。
- Hero 下不显示统计胶囊，关键词星图不显示“主题坐标”标题。
- 关键词由文章标题、章节和正文等权计算，不使用预设展示词或发布时间权重。
- 文章统一为左侧目录、右侧标题/标签/正文；文章源文件不自带样式、标题区或目录。
- 站点维护契约见 `.skills/notebook-site-maintainer/`，文章生成契约见 `.skills/kb-note-generator/`。

## 1. 品牌定位

- **中文名称**：低熵航线
- **英文辅助**：Low Entropy Course
- **Slogan**：在复杂系统中，持续构建秩序
- **站点属性**：个人笔记博客，不是技术媒体门户，也不是多人内容平台。
- **核心隐喻**：在复杂系统中持续观察、记录、复盘、抽象，把零散实践整理成更有秩序的个人知识航线。

### 1.1 首页气质关键词

- 克制
- 清晰
- 技术感
- 个人化
- 航线 / 轨道 / 星球 / 坐标
- 低熵 / 秩序 / 复杂系统

### 1.2 不采用的方向

- 不做 InfoQ 式多层导航。
- 不做登录、注册、投稿、活动推荐等媒体门户功能。
- 不把“专题”“归档”等当前没有明确内容结构的入口放入主导航。
- 不把主题按钮平铺在顶部导航里。

## 2. InfoQ 导航可借鉴点

InfoQ 的顶部导航有几个值得参考的设计点：

1. **品牌区明确**：左侧 Logo 是视觉锚点。
2. **主导航横向排列**：用户能快速理解站点主要内容入口。
3. **工具区右置**：搜索、发布、登录等功能与内容导航分区明确。
4. **信息密度可控**：虽然 InfoQ 内容很多，但通过分区降低混乱感。

低熵航线只保留这些原则，不复制它的二级频道、活动推荐和商业功能。

## 3. 导航栏设计

### 3.1 推荐最终结构

```text
┌──────────────────────────────────────────────────────────────────────┐
│  ◉ 低熵航线          首页    笔记    标签    关于          搜索   RSS   外观 ▾ │
└──────────────────────────────────────────────────────────────────────┘
```

### 3.2 分区结构

```text
左侧品牌区             中间内容导航                  右侧工具区
┌────────────┐      ┌──────────────────┐       ┌──────────────────┐
│ ◉ 低熵航线 │      │ 首页 笔记 标签 关于 │       │ 搜索 RSS 外观 ▾ │
└────────────┘      └──────────────────┘       └──────────────────┘
```

### 3.3 导航项说明

#### 首页

回到站点首页，承载品牌 Hero、最近笔记、标签入口和内容概览。

#### 笔记

主内容入口，代表个人技术笔记与阅读记录。

不再单独设置“专题”，因为当前并没有明确策展式专题页。如果未来出现稳定的主题集合，再考虑增加“专题”或“航线”。

#### 标签

保留现有标签系统，适合作为轻量内容发现方式。

#### 关于

保留。它不应做成传统简历页，而应是轻量站点说明页，解释：

- 低熵航线是什么。
- 为什么叫低熵航线。
- 这里记录什么。
- 写作原则是什么。
- 作者和订阅入口。

#### 搜索

保留为工具入口。桌面端可显示为“搜索”或“⌕ 搜索”，移动端可保留图标按钮。

#### RSS

保留为订阅入口。

#### 外观

替代当前平铺的“浅色 / 星夜 / 纸感”按钮。

```text
外观 ▾
├── 浅色
├── 星夜
└── 纸感
```

## 4. 主题切换设计

### 4.1 桌面端

右侧只显示一个下拉入口：

```text
外观 ▾
```

下拉菜单展开后显示：

```text
浅色
星夜
纸感
```

当前主题需要有选中态，例如：

```text
✓ 星夜
```

### 4.2 移动端

移动端顶部不平铺主题选项，只显示：

```text
◉ 低熵航线          搜索  ☰
```

菜单展开后：

```text
首页
笔记
标签
关于
RSS

外观
- 浅色
- 星夜
- 纸感
```

## 5. 首页 Hero 设计

### 5.1 推荐文案

```text
LOW ENTROPY COURSE
低熵航线
在复杂系统中，持续构建秩序
```

### 5.2 辅助说明文案

可选择下面其中一个作为 Hero 副说明：

```text
记录技术实践、阅读思考与长期观察，把零散经验整理成可回望的航线。
```

或更克制一点：

```text
关于工程实践、系统思考与个人知识整理的长期笔记。
```

推荐使用第一版，因为它与“航线”和“低熵”的品牌隐喻更贴合。

### 5.3 Hero 视觉元素

建议继续保留当前首页的网格、星图、坐标感，但将关键词从“星图 / atlas”转向“航线 / course / orbit”。

可以使用：

- 细线轨道
- 微弱星尘
- 坐标网格
- 航线箭头
- 品牌 mark 的星球轨道图形

不要使用过重的宇宙背景，以免个人博客显得像游戏或科幻专题站。

## 6. 首页内容区建议

首页不宜太复杂，建议保留 3 个区域。

### 6.1 最近笔记

展示最新文章列表，是个人博客最重要的内容入口。

标题建议：

```text
最近笔记
```

或更品牌化：

```text
近期航迹
```

推荐先使用“最近笔记”，更直观。

### 6.2 标签 / 关键词入口

保留关键词星图或标签云，但标题可从“关键词星图”改为更符合新品牌的表达。

候选：

```text
关键词航标
```

```text
主题坐标
```

```text
内容航标
```

推荐使用：

```text
主题坐标
```

原因：它既保留系统化感，又不会太生硬。

### 6.3 内容目录入口

如果当前首页已有目录或分类入口，可以保留，但不要命名为“专题”。

推荐标题：

```text
笔记目录
```

或：

```text
内容目录
```

推荐使用“笔记目录”，符合个人博客属性。

## 7. 关于页内容草案

如果本地实现时需要新增关于页，可以采用以下结构。

```markdown
# 关于低熵航线

低熵航线是我的个人笔记博客，用来记录技术实践、阅读思考与长期观察。

“低熵”代表一种持续整理复杂性的写作方式：从问题、实践、资料和经验中提炼结构，让混乱输入逐渐变成可复用的理解。

“航线”代表探索路径。这里的每篇笔记都不是终点，而是一次穿过复杂系统后留下的坐标。

## 这里记录什么

- 工程实践
- AI 工程
- Agent Runtime
- Rust 学习
- Git 与部署
- UI 设计
- 阅读笔记
- 工具链与效率

## 写作原则

- 记录实践后的复盘，而不是资料搬运。
- 关注可复用的心智模型，而不是零散技巧。
- 尽量把复杂问题讲清楚，整理成之后还能回看的结构。

## 订阅

可以通过 RSS 订阅更新。
```

## 8. 资源接入建议

已有品牌资源位于：

```text
site/assets/brand/
```

建议映射关系：

```text
atlas-light  -> low-entropy-route-lockup-light.svg
nebula-dark  -> low-entropy-route-lockup-dark.svg
paper-warm   -> low-entropy-route-lockup-paper.svg
导航 mark    -> low-entropy-route-mark.svg
```

导航栏建议优先使用 mark + 文字，不直接使用横版大 lockup。横版 lockup 更适合首页 Hero、社交分享图或品牌展示区。

## 9. 桌面端交互细节

### 9.1 导航高度

建议：

```text
64px - 72px
```

### 9.2 背景

浅色主题：

```text
半透明白 + 轻微 blur + 细边线
```

星夜主题：

```text
深蓝黑半透明 + 轻微星尘感 + 细边线
```

纸感主题：

```text
暖纸色半透明 + 轻纹理 + 暖色边线
```

### 9.3 当前页高亮

推荐使用短线，不使用大面积背景。

```text
首页
━━
```

或通过文字颜色 + 2px bottom border 表达。

### 9.4 外观下拉

要求：

- 点击按钮展开。
- 点击外部关闭。
- Esc 关闭。
- 支持键盘 Tab 访问。
- 当前主题显示选中态。
- 移动端放入菜单内部。

## 11. 实施步骤建议

### 第一步：更新站点基础信息

将站点标题和文案从旧名称更新为：

```text
低熵航线
Low Entropy Course
在复杂系统中，持续构建秩序
```

### 第二步：改造导航结构

将导航调整为：

```text
首页 / 笔记 / 标签 / 关于 / 搜索 / RSS / 外观
```

删除或不展示：

```text
专题 / 归档 / 平铺主题按钮
```

### 第三步：实现外观下拉

把当前主题切换按钮组收敛为一个下拉菜单。

### 第四步：更新首页 Hero

将 Hero 文案更新为低熵航线品牌表达。

### 第五步：调整首页区块标题

建议替换：

```text
关键词星图 -> 主题坐标
```

如有目录区：

```text
目录 -> 笔记目录
```

### 第六步：可选新增关于页

如果当前没有关于页，建议新增轻量关于页。

## 12. 验收标准

- 顶部导航只有一层。
- 导航中不出现“专题”和“归档”。
- 主题切换不再平铺，而是通过“外观”下拉选择。
- 首页品牌名称为“低熵航线”。
- 首页 slogan 为“在复杂系统中，持续构建秩序”。
- 品牌图形优先用于导航左侧 mark。
- 首页不引入登录、注册、投稿等平台型功能。
- 移动端导航不拥挤，菜单可展开访问主要入口。
