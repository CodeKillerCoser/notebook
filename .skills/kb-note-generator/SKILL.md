# kb-note-generator

基于对话历史自动提取知识点，生成结构化的 HTML 知识库笔记，并提交到 GitHub notebook 仓库。

## 目标仓库

- 仓库地址: `https://github.com/CodeKillerCoser/notebook`
- 分支: `main`
- 输出路径: 仓库根目录下，按主题分类的子目录

## 核心流程

### 1. 评估场景

当学习对话接近尾声或用户明确要求整理笔记时，触发该流程。

### 2. 提取知识点

扫描对话历史，识别以下类型的内容：

- **概念讲解** — 新学的编程概念、语法规则、技术原理
- **代码示例** — 实际编写或调试过的完整代码片段
- **错误/陷阱** — 遇到的问题及其根本原因和解决方案
- **配置/工具** — 开发环境、插件版本、构建工具相关的配置
- **版本差异** — 不同版本 API 的变化和迁移对照

### 3. 组织分类

将知识点按以下层级组织（详见 `references/output_structure.md`）：

```
主题根目录/
├── index.html                  # 目录索引，列出所有文档及概要
├── 01-基础入门/
│   ├── 话题1.html
│   └── 话题2.html
├── 02-语法进阶/
│   └── ...
├── 03-工具链与生态/
│   └── ...
└── 04-综合案例/
    └── ...
```

分类原则：
- 从基础到进阶，按学习路径排序
- 每个一级目录下放 2-5 个文档，保持扁平
- 目录名用两位数字前缀控制排序
- 综合案例放最后，作为知识串联

### 4. 生成笔记

笔记格式遵循 HTML 模板（见 `references/output_template.md`）：
- 每个文档一个独立的 HTML 文件
- 包含内联 CSS 样式，确保独立渲染效果良好
- 代码块使用 `<pre><code>` 标签并标注语言类型
- 错误/正确示例分别标记 ❌ / ✅
- 文档间用相对路径交叉引用

生成完所有笔记后，创建或更新 `index.html` 索引文件。

### 5. 提交到仓库

按 `references/github_workflow.md` 中的流程操作：
1. 克隆仓库到临时目录
2. 复制 HTML 笔记文件到对应路径
3. `git add` → `git commit` → `git push`
4. 清理临时目录

## 注意事项

- 提取知识点时优先扫描 `note.md` 等已有的笔记文件
- 引用的代码必须来自对话中实际编写的代码，不要编造
- 提交信息格式：`docs(<主题>): <简短描述>`
- 所有 Git 操作限于 notebook 仓库，不修改用户其他仓库
